The framework is a synthesis; these are the primary sources. Start with the practitioners, then the official engineering writing, then the maturity-model literature.

## Practitioners

- **Boris Cherny** — [Best practices for Claude Code](https://code.claude.com/docs/en/best-practices) · [How Boris Uses Claude Code](https://howborisusesclaudecode.com/)
- **Andrej Karpathy** — ["Software Is Changing (Again)" / Software 3.0](https://www.ycombinator.com/library/MW-andrej-karpathy-software-is-changing-again) · the original "vibe coding" framing and its 2026 "agentic engineering" retrospective
- **Thariq Shihipar** — [Claude Code is All You Need](https://x.com/trq212/status/1944877527044120655) · ["HTML is the new markdown"](https://www.lennysnewsletter.com/p/html-is-the-new-markdown-how-anthropic)
- **Simon Willison** — [Agentic Coding](https://simonwillison.net/2025/Jun/29/agentic-coding/)
- **Armin Ronacher** — [Agentic Coding Recommendations](https://lucumr.pocoo.org/2025/6/12/agentic-coding/)
- **Geoffrey Huntley** — [Ralph Wiggum as a software engineer](https://ghuntley.com/ralph/)
- **Thorsten Ball** — [How to Build an Agent](https://ampcode.com/notes/how-to-build-an-agent)
- **Hamel Husain** — [Your AI Product Needs Evals](https://hamel.dev/blog/posts/evals/)
- **Steve Yegge** — [Revenge of the Junior Developer](https://sourcegraph.com/blog/revenge-of-the-junior-developer)

## Maturity-model & metrics frameworks

- [Capability Maturity Model](https://en.wikipedia.org/wiki/Capability_Maturity_Model) — the 5-level shape
- [DeepMind: Levels of AGI](https://arxiv.org/abs/2311.02462) — performance × generality, with autonomy as a separate axis
- [SAE J3016 levels of driving automation](https://www.sapien.io/glossary/definition/sae-levels-of-automation) — division of responsibility as the basis for levels
- [Factory.ai: Agent Readiness](https://factory.ai/news/agent-readiness) — the binary, file-existence codebase rubric
- [DORA 2025 AI Capabilities Model](https://dora.dev/dora-report-2025/) — the team/org layer
